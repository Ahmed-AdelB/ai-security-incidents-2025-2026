## LaTeX Compilation Workflow

### Pre-Compilation Checks

- [ ] All \cite{} references have corresponding entries in bibliography.bib
- [ ] All \ref{} and \label{} cross-references are consistent
- [ ] All \input{} and \include{} file paths are correct
- [ ] All image paths in \includegraphics{} are correct
- [ ] No undefined command macros (\newcommand{} all defined)
- [ ] Section hierarchy is logical (\section → \subsection → \subsubsection)

### Compilation Sequence

```bash
# Step 1: First pass (structure and references)
pdflatex -interaction=nonstopmode paper.tex

# Step 2: Bibliography resolution
bibtex paper

# Step 3: Second pass (integrate bibliography)
pdflatex -interaction=nonstopmode paper.tex

# Step 4: Third pass (finalize cross-references)
pdflatex -interaction=nonstopmode paper.tex

# Step 5: Verify output
ls -lh paper.pdf
```

### Expected Output Files After Compilation

- [ ] **paper.pdf** — Final compiled PDF (for preview before arXiv submission)
- [ ] paper.aux — Auxiliary file (LaTeX internal)
- [ ] paper.log — Compilation log (check for warnings/errors)
- [ ] paper.bbl — Processed bibliography (generated by bibtex)
- [ ] paper.blg — Bibliography log

### Troubleshooting Common Compilation Errors

| Error | Cause | Fix |
|-------|-------|-----|
| `Undefined control sequence \cite{...}` | Missing or misspelled citation | Check bibliography.bib entries, ensure bibtex ran successfully |
| `LaTeX Warning: Reference 'fig:...' on page X undefined` | Missing \label{} or typo in \ref{} | Add \label{} after \caption{} in figure environments |
| `File not found: figures/...` | Incorrect image path | Use relative paths from paper.tex directory |
| `pdflatex: command not found` | LaTeX not installed | Install texlive-latex-base (Linux/Mac) or MiKTeX (Windows) |
| `bibtex: command not found` | BibTeX not installed | Install texlive-bibtex or included in full texlive distribution |

## Abstract Section

**Title (Recommended):**
AISecIncidents-2025-26: A Dataset of AI Security Incidents with EU AI Act Compliance Mapping

**Abstract (copied from ARXIV_ABSTRACT.md):**

We present AISecIncidents-2025-26, a dataset of 55 AI security incidents spanning January 2025–April 2026, of which 12 are verified through multi-source public confirmation. Alongside the dataset, we conduct a systematic static analysis of 1,332 TypeScript source files from a leaked production AI agent (Claude Code v2.1.88). Our analysis reveals 23 AST-based security checks in the BashTool execution engine, a novel Semantic Desync vulnerability class arising from non-deterministic LLM intent mapped to deterministic parser execution, and UNDERCOVER MODE—a 57,000-word obfuscation instruction set. We map findings to EU AI Act compliance requirements (Articles 9, 13, 15) and release a multi-label classification benchmark (aisec-euaiact-v1.0, n=12 ground truth labels).

**Key contributions:** (1) First large-scale open dataset correlating real-world LLM security incidents with regulatory compliance requirements; (2) Novel static analysis framework for AI agent security assessment with 23 AST-based checks and Semantic Desync vulnerability classification; (3) Discovery and analysis of UNDERCOVER MODE, a 57,000-word hidden instruction set in Claude Code; (4) Standardized incident taxonomy with confidence scoring enabling quantitative benchmarking of incident response effectiveness across the AI industry.
